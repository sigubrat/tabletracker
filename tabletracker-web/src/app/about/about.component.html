TableTracker

A website for tracking if the table is available

The application consists of three parts:

  • An Arduino with a motion sensor
  • An ASP.NET Web API
  • An Angular application to interact (you're looking at it)

Workflow:

  1. The Arduino sends an HTTP-request to the Web API whenever it detects movement.
  2. The Web API stores if the table is available and the timestamp of the last update.
  3. The web-server contacts the Web API through a GET-request and displays the result.
  4. If too much time has passed since the last detected movement, the server will reset the state to 'Available'.
